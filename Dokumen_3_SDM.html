1.3 Deskripsi Peran dan Kompetensi

Peran Tanggung Jawab Kompetensi yang Dibutuhkan
Project Manager / Scrum Master • Mengelola keseluruhan proyek
• Memfasilitasi ceremony Scrum (sprint planning, daily standup, review, retrospective)
• Menghilangkan hambatan (impediments)
• Komunikasi dengan stakeholder
• Monitoring progress dan risiko 		• Pengalaman PM minimal 2 tahun
• Sertifikasi Scrum (CSM/PSM) diutamakan
• Kemampuan komunikasi & leadership
• Familiar dengan tools manajemen proyek (Jira, Trello)
System Analyst / Product Owner • Analisis dan dokumentasi kebutuhan sistem
• Mengelola Product Backlog
• Memprioritaskan fitur berdasarkan value bisnis
• Menjembatani komunikasi antara tim dan stakeholder
• Acceptance criteria untuk setiap user story 		• Pengalaman SA minimal 2 tahun
• Pemahaman proses bisnis pengadaan barang
• Kemampuan membuat UML, ERD, flowchart
• Kemampuan analitis dan dokumentasi
UI/UX Designer • Desain wireframe dan prototype
• Membuat desain visual (mockup high-fidelity)
• User research dan usability testing
• Membuat design system / style guide 		• Pengalaman UI/UX minimal 1 tahun
• Mahir Figma / Adobe XD
• Pemahaman responsive design
• Pemahaman UX principles
Backend Developer (2 orang) • Pengembangan server-side logic dan API
• Desain dan implementasi database
• Implementasi business logic (approval workflow, kalkulasi)
• Integrasi file upload
• Unit testing 		• Pengalaman backend minimal 2 tahun
• Mahir bahasa pemrograman server-side (PHP/Java/Node.js/Python)
• Mahir RDBMS (MySQL/PostgreSQL)
• Pemahaman RESTful API
• Pemahaman keamanan aplikasi
Frontend Developer • Pengembangan antarmuka pengguna
• Implementasi desain dari UI/UX designer
• Integrasi dengan backend API
• Responsive design implementation 		• Pengalaman frontend minimal 1-2 tahun
• Mahir HTML, CSS, JavaScript
• Mahir framework frontend (React/Vue.js/Angular)
• Pemahaman state management
QA Engineer / Tester • Membuat test plan dan test cases
• Functional testing, integration testing
• Regression testing setiap sprint
• UAT coordination
• Bug tracking dan reporting 		• Pengalaman QA minimal 1 tahun
• Pemahaman testing methodology
• Familiar dengan bug tracking tools (Jira, Bugzilla)
• Kemampuan automation testing diutamakan
Database Administrator (rangkap) • Desain schema database
• Optimasi query dan indexing
• Backup dan recovery plan
• Data migration (jika ada) 		• Pengalaman DBA minimal 1 tahun
• Mahir SQL dan database design
• Pemahaman normalisasi dan optimasi
• Dirangkap oleh Backend Developer 1

1.4 Matriks RACI (Responsible, Accountable, Consulted, Informed)

Matriks RACI menunjukkan tanggung jawab setiap peran terhadap aktivitas proyek:

📌 Keterangan RACI:
R = Responsible (mengerjakan)   A = Accountable (bertanggung jawab)   C = Consulted (dikonsultasi)   I = Informed (diinformasikan)
Aktivitas PM/SM SA/PO UI/UX BE Dev FE Dev QA
Sprint Planning R A C C C C
Analisis Kebutuhan I R/A C C I I
Desain Database (ERD) I A I R I I
Desain UI/UX I C R/A I C I
Backend Development I C I R/A I I
Frontend Development I C C C R/A I
Testing I C I C C R/A
Sprint Review R A R R R R
UAT Coordination A R I C C R
Deployment A I I R R C
Training User A R C I I I
Dokumentasi A R C R R R
